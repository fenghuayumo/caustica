裸眼 3D 光学映射交互演示

按这个顺序看：子像素是什么，光学层怎么分光，左右眼落到哪个 view，串扰怎么混，calibration 为什么能补误差。

核心映射：screen(x, y, subpixel) -> optics -> outgoing angle -> left/right eye
view index：不是屏幕坐标，而是出射方向编号。
head tracking：估计左右眼坐标，决定当前应看哪个 view。
calibration：测真实硬件映射，把理论输出修正到真实输出。

1显示面板：像素由 RGB 子像素组成

裸眼 3D 的映射经常细到子像素级别。一个逻辑像素通常包含 R、G、B 三个发光/透光小条。

对普通 2D 渲染，pixel 是颜色单元；对裸眼 3D，subpixel 还是空间采样点和角度采样点。

2一个透镜单元放大：不同子像素被送到不同方向

把一个柱状透镜下面的一排子像素放大看：左侧子像素会偏向一个方向，右侧子像素会偏向另一个方向。

同一块屏幕不是整面朝你发一张图；光学层把不同子像素分配给不同观看方向。多视角裸眼 3D 就是在这些方向上放不同相机视图。

3head tracking：左右眼只是两个空间采样点

下面这条带子是观察空间里的 view zone。眼睛位置变了，就会命中不同 view；系统要实时跟着改输出。

4interlacing：把多视角图像写回子像素

这里用小方格模拟 framebuffer。蓝色代表左眼当前 view 的内容，红色代表右眼当前 view 的内容，灰色是其他方向视角。

渲染侧的任务是：生成 V0、V1、V2...，然后按 calibration 得到的映射表，把每个 view 填到对应子像素。

5lens slant：为什么透镜常常是斜的

如果透镜完全竖直，分辨率损失主要压在水平方向；倾斜后，view index 会随 y 发生偏移，把采样压力分摊到二维。

6crosstalk matrix：一个眼睛会混入相邻 view

这就是重影来源。串扰越高，左眼越容易看到一点右眼/邻近方向的内容。

7calibration：测真实硬件，然后反向补偿

理论上 View 3 应该射到某个方向，但透镜贴合有偏移时会整体漂。calibration 的工作就是测出漂移，并在软件写图时反向偏移。

这和多视角重建里的相机估计很像：不是相信设计参数，而是用观测数据拟合真实映射，再让渲染输出服从这个映射。
